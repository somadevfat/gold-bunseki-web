### Husky

導入
bun add -d husky @commitlint/cli @commitlint/config-conventional

Husky初期化
bunx husky init

Commitlintの設定ファイルを作成する
echo "export default { extends: ['@commitlint/config-conventional'] };" > commitlint.config.js

コミット実行時のフック（commit-msg）を設定する
echo "bunx --no -- commitlint --edit \$1" > .husky/commit-msg

エンティティーを定義
まとまりを。あとからdbを変えても大丈夫なように。
mkdir -p apps && cd apps && bun create next-app@latest frontend --yes

# 1. API(Go)の箱を作って初期化

mkdir -p apps/api
cd apps/api
go mod init gold-vola-bunseki/api
touch main.go

# 2. Analytics(Python)の箱を作って初期化

cd ../../
mkdir -p apps/analytics
cd apps/analytics
python3 -m venv venv
touch main.py requirements.txt
